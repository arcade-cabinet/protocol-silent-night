## Critical Rules

1. **Holidaypunk identity is non-negotiable.** Every visual, audio, and mechanical choice must read as festive-industrial-menacing. Not generic techno, not cute Christmas, not sterile sci-fi.
2. **Death is the game.** The difficulty formula scales superlinearly. Upgrades are pressure relief valves, not permanent godmode. Every run ends in being overwhelmed. Level 999 is possible but EARNED.
3. **You play as presents.** The player roster is 25-50 anthropomorphic present variants (box + arms + face + weapon + gear). Current characters (Elf, Santa, Bumble) are ENEMIES, not playables.
4. **No hardcoded wave tables.** Wave content is generated by a PRNG-seeded formula where level acts as a force multiplier on speed, composition, pattern, frequency, and pressure levers.
5. **Max 200 LOC per GDScript file.** Enforced by `.claude/settings.json` hook. Decompose large scripts.
6. **Do not upgrade Gaea or gdUnit4 casually.** Read `docs/DEPENDENCY_POLICY.md` first.

## Wave Generation Formula

Waves are NOT a static table. The formula accepts `(run_seed, level)` and outputs:

| Parameter | Description |
|-----------|-------------|
| `spawn_rate` | Enemies per second, scales with level and swarm_weight |
| `composition` | Enemy type mix, shifts by level and seed personality |
| `hp_scale` | Enemy durability, superlinear growth |
| `speed_mult` | Enemy speed, grows to eventually exceed dash speed |
| `pattern_type` | scatter / ring / wedge / flanking / spiral |
| `burst_chance` | Probability of burst spawn events |
| `boss_threshold` | Level at which boss variants appear |

The **run seed** generates a pressure profile (6 weights) that determines which axes this run emphasizes. The **level** force-multiplies along those axes. Result: every run feels different, difficulty always wins eventually, but exceptional play can push absurdly far.

## Present Character Roster

Each present is procedurally defined from:
- **Box shape**: tall/wide/cube/tiny (communicates stat archetype)
- **Color scheme**: holidaypunk palette (red, cyan, gold, green, black, purple)
- **Face**: expression sprite (determined, angry, cheerful, stoic, manic)
- **Arms**: stick/beefy/mechanical
- **Weapon**: defines primary combat identity
- **Accessory**: hat, bow, ribbon, tag — visual flair

25-50 variants, each with distinct name, weapon, and stat profile. Unlocked through achievement milestones, not arbitrary wave numbers.

## Enemy Roster

Enemies include both the original types AND the former playable characters:
- **Grunt (Grinch-Bot)**: baseline pressure
- **Rusher (Reindeer-Rusher)**: fragile, fast
- **Tank (Gingerbread Golem)**: slow, durable
- **Elf (Cyber-Elf)**: fast, rapid-fire stream — now hostile
- **Santa (Mecha-Santa)**: durable shotgun bruiser — now hostile
- **Bumble (The Bumble)**: heavy projectile specialist — now hostile
- **Krampus-Prime**: boss encounter, scales per formula

## Code Standards

- Max 200 LOC per `.gd` file (hook-enforced)
- No monolithic scripts — decompose by responsibility
- `.tres` resources for structured data, not nested Dictionaries
- Declarations (JSON) for content values, GDScript for behavior
- No `load()` for static data — use Resources or autoloads
